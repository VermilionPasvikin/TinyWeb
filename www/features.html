功能介绍

TinyWeb 服务器引擎

HTTP 协议支持

特性 支持情况 说明
HTTP/1.0 ✓ 支持 完整支持HTTP/1.0协议
HTTP/1.1 ✓ 支持 支持HTTP/1.1但不支持持久连接
GET 方法 ✓ 支持 获取静态文件和CGI动态内容
POST 方法 ✓ 支持 通过stdin传递请求体给CGI
HEAD 方法 ✓ 支持 只返回响应头
持久连接 ✗ 不支持 每个请求后关闭连接

架构特性

  • 多线程模型：8个工作线程
  • 连接队列：最多缓存32个待处理连接
  • 生产者-消费者模式：使用信号量同步
  • 信号处理：优雅的SIGCHLD处理避免僵尸进程

静态文件服务

支持多种MIME类型的静态文件：

HTML (.html, .htm)
CSS (.css)
JavaScript (.js)
图片 (.jpg, .jpeg, .png, .gif, .ico, .svg)
文本 (.txt)
JSON (.json)
PDF (.pdf)

CGI 动态内容

CGI程序特性：

  • 自动编译：cgi-bin/*.c文件在编译时自动生成可执行文件
  • 环境变量：REQUEST_METHOD, QUERY_STRING, CONTENT_LENGTH, CONTENT_TYPE
  • 输入输出：POST数据通过stdin传入，响应通过stdout输出
  • 进程隔离：每个CGI请求在独立子进程中执行
  • 无超时限制：CGI进程可以运行任意时长

管理面板

可选的管理面板功能（通过--admin-port启动）：

  • 实时仪表盘：2秒自动刷新
  • 连接统计：当前连接数、总请求数、错误数
  • 运行时信息：运行时长、吞吐量
  • JSON API：/__admin/status, /__admin/metrics, /__admin/shutdown

示例访问：http://localhost:9090/__admin/（需启动时指定端口）

TinyProxy 反向代理

负载均衡

  • 算法：轮询（Round-robin）
  • 后端数量：最多支持8个后端服务器
  • 权重：所有后端权重相同

健康检查

  • 检查间隔：每3秒一次
  • 检查方式：尝试TCP连接到后端端口
  • 自动故障转移：不健康的后端自动跳过
  • 自动恢复：恢复健康的后端自动加入轮询

请求处理

  • 头部注入：自动添加X-Forwarded-For头
  • 双向中继：客户端和后端之间的双向数据传输
  • 超时控制：30秒连接超时
  • 透明代理：保持原始请求内容

管理接口

代理统计API：

GET /__admin/backends
返回所有后端的状态和请求计数

GET /__admin/status
返回代理服务器的运行状态

GET /__admin/metrics
返回详细的性能指标

使用示例

启动TinyProxy：
./TinyProxy 8080 localhost:8081 localhost:8082 localhost:8083 --admin-port 9091

配置说明：
- 监听端口：8080
- 后端服务器：8081, 8082, 8083
- 管理端口：9091

命令行示例

测试GET请求

curl http://localhost:8080/
curl http://localhost:8080/cgi-bin/hello
curl http://localhost:8080/cgi-bin/info?test=value

测试POST请求

curl -X POST http://localhost:8080/cgi-bin/calc \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "num1=10&operation=add&num2=5"

测试HEAD请求

curl -I http://localhost:8080/

访问管理面板

curl http://localhost:9090/__admin/status
curl http://localhost:9090/__admin/metrics
